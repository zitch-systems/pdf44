# PDF44 — Platform Audit Round 6

_Audited: 2026-06-30 · Scope: web SPA (`index.html`), PWA infra (`sw.js`, `_headers`), SEO (`sitemap.xml`), Supabase backend + migrations, admin portal, React Native mobile app, CI._

This round re-audited the whole platform for **net-new correctness, privacy, and freshness bugs** (rounds 1–5 are in `AUDIT.md` / `AUDIT-2026-06-28.md`). Confirmed fixes were applied; by-design tradeoffs and low-value edge cases are listed as open so they aren't re-litigated.

---

## Fixed this round

### CRITICAL / HIGH

**R6-1 HIGH — Camera left running after navigating away from Scan-to-PDF**
- `index.html` — `scanStream` (a `getUserMedia` MediaStream) was only stopped inside `processScan()`. Starting the scanner then clicking any other tool or pressing Back left the webcam / phone camera **active and recording-indicator lit indefinitely** (`route()` cleared `state.files` but never released the stream).
- Fix: added `stopScanner()` (stops all tracks, detaches `srcObject`, clears `scanPages`/`state.scanCount`/thumbnails) and called it from `route()` on every navigation and from `processScan()` after the PDF is produced.

**R6-2 HIGH — In-place editing tools double-apply on a second "Process" click**
- `index.html` — `processEdit`, `processWatermark`, `processPageNumber`, `processSign`, `processInsertBlank`, `processHeader`, `processBates`, and `processEditPdf` all drew onto the **shared cached `f.pdfDoc`** and saved it, never working on a copy. `showResult()` leaves the file loaded with Process still enabled, so clicking Process twice produced a doc with **two watermarks / stacked page numbers / duplicated signatures / extra blank pages each click**. `processRotate` was already fixed this way in round 5 (MUT-1); the other eight editors had the same defect.
- Fix: added a one-line `refreshDoc(f)` helper that reloads `f.pdfDoc` from the file's retained original `arrayBuffer` at the start of each editor, so every run begins from a pristine document. Robust even if a prior run errored mid-draw. Read-only/copy-based tools (merge, split, extract, remove, organize, background, compress) were already correct and untouched.

### MEDIUM

**R6-3 MEDIUM — `config.js` (and `pdf44-account.js`) pinned in browser cache for 30 days, defeating the SW's freshness intent**
- `_headers` — the generic `/*.js` rule stamped `Cache-Control: immutable, max-age=2592000` on **all** JS, including `config.js` (live entitlement flags + Supabase/Paystack public keys + ad kill-switch) and `assets/pdf44-account.js` (billing logic). `sw.js` deliberately fetches `config.js` network-first, but the browser HTTP cache still served an immutable copy, so a rotated key or flipped kill-switch would not reach returning visitors for 30 days.
- Fix: added explicit, more-specific `/config.js` (`no-cache, must-revalidate`) and `/assets/pdf44-account.js` (`max-age=0, must-revalidate`) header rules ahead of the wildcard.

**R6-4 MEDIUM — Service worker served `pdf44-account.js` cache-first (stale billing logic for installed PWAs)**
- `sw.js` — the SW excluded `config.js` (network-first) and Supabase/Paystack (never-cache) to keep entitlement fresh, but `assets/pdf44-account.js` (the live `isPremium()`/subscription/checkout controller) fell through to the generic cache-first branch, so a shipped billing fix wouldn't reach installed-PWA users until a cache-name bump.
- Fix: extended the always-fresh (network-first) matcher to cover `pdf44-account.js`; bumped `pdf44-v7` → `pdf44-v8` to evict any already-cached copy on existing clients.

### LOW

**R6-5 LOW — `formatBytes()` returned "1.00 undefined" for ≥1 TB**
- `index.html` — the `sizes` table stopped at `GB`, so `Math.floor(log/log(1024)) ≥ 4` indexed out of bounds. Fix: added `TB`/`PB` and clamped the index.

**R6-6 LOW — Sitemap advertised 5 non-canonical `?tool=` duplicate URLs**
- `sitemap.xml` — `/?tool=merge|split|compress|pdf2word|sign` load the homepage with a query param (canonical → `/` statically, or the clean slug after JS), so submitting them to crawlers created duplicate-content noise. The clean-slug pages already cover every tool. Fix: removed the 5 legacy entries (the old links still resolve for bookmarks via the SPA). Sitemap now 196 URLs (CI floor is 50).

**R6-7 LOW — Dead `onLayout={() => navTab}` no-op in mobile Settings**
- `mobile/src/screens/Settings.tsx` — a spacer `<View>` referenced `navTab` without calling it. Fix: removed the dead node and dropped the now-unused `navTab` from the destructure.

## Verified SAFE (reported by an audit pass but mitigated by code I confirmed)

- **Admin portal has no server-side auth → relies on RLS.** Verified the RLS in migrations 0001/0004/0005/0006 is correct: `*_select_own_or_admin` + admin-only `*_admin_all` policies on `profiles`/`subscriptions`/`payments`, public-read/admin-write on `site_settings`, and the `guard_profile_role` trigger that freezes `role`/`email` for non-admin clients. A non-admin opening `/admin` cannot read others' data or self-promote.
- **Admin "Grant premium" could create duplicate active subs.** Mitigated by the `uniq_active_sub_per_user` partial unique index (migration 0001) — a second active row is rejected by the DB.
- **Admin revenue/MRR currency mixing.** Latent only: the app charges a single currency (`config.js` plans are NGN; `paystack-initialize` uses one `PAYSTACK_CURRENCY`), so `money()`/`moneyMajor()` are correct in practice. Would misreport only if mixed-currency payments ever exist.
- **Backend payment flow** (`paystack-verify` ownership/amount from the `payments` row, webhook HMAC + idempotency ledger + 500-on-DB-error, `config.toml` `verify_jwt` pins) — re-verified, no new issues.

## Open / by-design (not changed)

| Finding | Why left as-is |
|---|---|
| `localStorage.pdf44_prem` paywall bypass; IP-based quota spoofing | Inherent to client-side-only enforcement; files are processed locally and never uploaded. Server quota is a soft deterrent. (Acknowledged in prior rounds.) |
| `download-quota` fails **open** on HTTP error | Deliberate: a metering blip must never block a legitimate free/paying download. |
| CSP `connect-src 'self' https:` / `frame-src https:` | Intentionally broad to allow rotating ad-creative CDNs; with `'unsafe-inline'`/`'unsafe-eval'` already required, tightening adds little. |
| HTML→PDF `innerHTML = userHtml` (self-XSS only) | User renders their own pasted HTML; sandboxing into an iframe is a rendering-fidelity change (html2canvas) deferred to a feature branch. |
| Fill-PDF field-id collisions on names differing only in punctuation | Rare; needs a unique-id scheme. Documented. |
| Mobile: Edit-screen "Sign" uses a placeholder ink path; Search/Home use static `RECENTS`/`SEED_FILES` | The mobile app is an explicit preview build; honest "preview mode" tooling. Tracked for the real-pipeline effort (prior C4). |
| 573 KB inline script / 212× prerender duplication | Structural; needs the build-pipeline split from round 5's roadmap. |
